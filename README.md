# Sudoku Solver
Play project created to learn the Elixir language.
The solver is rule based and acts as a normal player would by noting down possible values for the cells
and applying logic such as hidden-pairs to solve the puzzle. The intention was to complete a small project
in the beginning of learning the language and perhaps come back later with more Elixir skills and refactor it.

### Why?
I read the book Principles by Ray Dalio and it sort of stuck with me how they used
the computer as a way of enhancing the human by translating their thought patterns
into algorithms. An algorithm can run 24/7, has no feelings and is a lot better
than humans at performing certain tasks.

Now, I'm sure that there are better algorithms to solve a sudoku but the point with
this project was to first learn a new language but mostly to try and encode, literally,
my way of solving sudoku puzzles. Nothing fancy, just how I solve them.
I'm also aware that this algorithm might not be able to solve all sudoku puzzles.
For example, Arto Inkala created, according to him, in 2012 the worlds hardest sudoku.
I attempted to solve it but was unable to. So, any puzzles I'm not personally
able to solve, this algorithm wouldn't be able to solve either.

To be honest, it feels pretty great to see this program work as intended. One of the
advantages is that the algorithm is easy to explain and I can put my full trust 
behind it. The point is not that sudoku itself is an important task to solve but that
being able to translate your thoughts into an algorithm and see it work gave me
another perspective on what I can do with computers in general and how they are
inherently an amazing tool.

### Other languages
Perhaps one could implement this a lot easier using a logic language such as Prolog.

Maybe an machine learning approach could be used although explainability is always
an issue with those solutions which results in lack of trust.
